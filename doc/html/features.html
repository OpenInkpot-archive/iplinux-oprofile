pid/pgrp filter

There are situations where you are only interested in the profiling results of a particular running process, or process group. You can enable the optional filter at build time, and set the pid/pgrp values via the --pid-filter and --pgrp-filter options to op_start, or by setting the relevant sysctls as mentioned in the section called sysctl tree in Chapter 2.

Note: Enabling filtering can have an impact on the performance of the profiler even when a filter is not set (numbers not yet available).
